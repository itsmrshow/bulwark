# Contributing

Thanks for your interest in contributing to Bulwark!

## Getting Started

1. Fork the repo and create a feature branch.
2. Keep changes focused and include tests when reasonable.
3. Run the checks before opening a PR.

## Development

```bash
go mod download
make build
make test
make lint
```

## Pull Requests

- Describe the problem and the solution.
- Include any relevant screenshots or logs.
- Note any breaking changes or migrations.

## Reporting Issues

Please include:

- Environment details (OS, Docker version, Bulwark version).
- Steps to reproduce.
- Expected vs. actual behavior.
- Logs or screenshots if available.
